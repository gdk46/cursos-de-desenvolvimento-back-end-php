Git - Primeiros passos

O Git é um sistema de controle de versões do seu software (projeto)
que te pemite acompanhe o ciclo de vida do projeto e o versionamento.

Git - fluxo de trabalho

Seus repositórios local baseiase em três "árvores" mantidas pelo .git.
A Working Directory (contém os arquivos novos), Index (uma área temporária),
HEAD (confirmação de envio).
Nessas "árvores" existe galhos (termo técnico Branches) que são utilizados para
desenvolver funcionalidades isoladas umas das outras.

Download

Para que o git funcione é necessário fazer o download em algumas distritos.
Veja se ele está instalado em sua máquina com o comando:

git --version

Se estiver intalado irá mostrar algo similar a isso:

git version 2.28.0

Click em um dos link para baixar o git caso não esteja instalado.

Mac Windows Linux

Iniciando versionamento do projeto

Comando: git init

O git init ele inicializa um repositório local criando ou tornar
um projeto já existente em versionável.

Sytaxe: git init

Se conectanto com um sistema remoto

Comando: git remote add e git clone

O git remote add: adiciona um repositório local em um repositório remoto

Sytaxe: git remote add [repositório]

O git clone: clona um repositório remoto para um repositório local

Sytaxe: git clone [repositório]

Adicionando meu projeto em uma Branches
com alegria

Comando: git add . ou git add [arquivo]

O git add . : adiciona todos os arquivos a uma Branches.

Sytaxe: git add .

O git add [arquivo]: adiciona um arquivo expecífico a uma Branches.

Sytaxe: git add [arquivo]

Compromentendo um arquivo ao ramo

Comando: git commit

O git commit: cria um novo compromentimento ao conteúdo
atual do índice e a mensagem de log fornecida descrevendo as mudanças,
sendo assim, cria uma hierarquia no cabeçalho (HEAD)

Sytaxe: git commit -m "sua mensagem"

Enviando alterações para o repositório remoto

Comando: git push ou git push origin master

O git push: é utilizado para enviar do projeto

Sytaxe: git push

O git push origin master: é utilizado para enviar as mudanças
do projeto pela primeira vez

Sytaxe: git push origin master

Atualizando repositório local

Comando: git pull

O git pull: é utilizado para atualizar o repositório local
em relação ao repositório remoto

Sytaxe: git push
